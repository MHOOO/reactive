Getting Started with reactive-web

  1. Choose an existing Lift project or create a new one

  2. Make sure you have the Sonatype maven repository configured

    For instance, if you're using sbt 0.7.x, you should have in your project file, e.g., project/build/Project.scala:

      val sonatype_snapshot = "Sonatype snapshots" at "http://oss.sonatype.org/content/repositories/snapshots"

    For sbt 0.10, that would be:

      resolvers += "Sonatype snapshots" at "http://oss.sonatype.org/content/repositories/snapshots"

  3. Add reactive-web as a dependency

    For instance, if you're using sbt 0.7.x, in project/build/Project.scala (or whatever your project file is called), for example:

      val reactive_web = "cc.co.scala-reactive" %% "reactive-web" % "0.0.1-SNAPSHOT"

    For sbt 0.10, that would be:

      libraryDependencies += "cc.co.scala-reactive" %% "reactive-web" % "0.0.1-SNAPSHOT"

    Sbt 0.10 verifies checksums by default. However, the repository's checksums may not be up to date. To disable the check, add this:

      checksums := Nil

  4. Optionally enable Comet support

    In Boot.scala:

      Reactions.initComet

    Once on each page, before a snippet that uses Comet support (may be in individual templates or in your surround template, it doesn't matter):

      <div class="lift:reactive"></div>

  5. Start coding!

    • You will probably want to import reactive._ and reactive.web._
    • Snippets may need to extend Observing
    • See the rest of this site for more information
